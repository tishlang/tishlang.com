## Create a Tish file

Create `main.tish`:

```javascript
fn greet(name) {
    return `Hello, ${name}!`
}

console.log(greet("Universe"))
```

## Run with the interpreter

```bash
tish run main.tish
```

Output: `Hello, World!`

To try code interactively with multi-line input, tab completion, and history, use the [Interactive REPL](/getting-started/repl/): run `tish` with no arguments **in a normal terminal** (stdin is a TTY).

## Run from stdin

You can pipe a program instead of using a file (like **`node -`** or **`bun`** with a pipe):

```bash
echo 'console.log("Its")' | tish
echo 'console.log("a small")' | tish run -
echo 'console.log("universe")' | tish -
```

When stdin is **not** a terminal, bare **`tish`** reads the full script from stdin and runs it once. See [Installation](/getting-started/installation/) for the full table of variants.

## Compile to native binary

```bash
tish compile main.tish -o wasup
./wasup
```

Same output. The compiled binary has no runtime dependency on Tish.

For backend options (Rust vs Cranelift), see [Native Backend](/reference/native-backend/). For WebAssembly (browser or Wasmtime), see [WASM Targets](/reference/wasm-targets/).

## HTTP server (requires `http` feature)

Create `server.tish`:

```javascript
fn handleRequest(req) {
    return { status: 200, body: `Hello, ${req.path}` }
}

serve(8080, handleRequest)
```

Run (the tish binary must be built with the http feature):

```bash
cargo run -p tishlang--features http -- run server.tish
```

Or compile to a standalone binary:

```bash
tish compile server.tish -o server --feature http
./server
```

Then visit `http://localhost:8080/`.
